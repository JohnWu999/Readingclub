01

核心洞察：为什么从情绪切入

"问题驱动式阅读"的底层逻辑是相通的——不是让用户去适应内容，而是让内容来回应用户当下的状态。 —— 犟爸

书房现有的产品架构已经非常清晰：用户带着困惑入群 → AI 标记相关内容线索 → 好奇心驱动深度阅读。这是一个从"问题"出发的漏斗。但"问题"和"情绪"之间，还有一层更前置、更频繁、更柔软的入口。

一个妈妈在晚上十点打开书房，她可能还没有形成清晰的"问题"，但她有一个强烈的"状态"：今天又被孩子磨到没脾气了。这时候她不需要一本书的目录，她需要被接住。

🔑 核心判断：情绪入口 vs 问题入口
维度 问题入口（已有） 情绪入口（新增）
触发时机 用户有明确困惑时 用户有模糊感受时
用户状态 "我想解决 X" "我今天好累/好烦/好委屈"
内容形态 章节阅读 + AI 教练 树洞倾诉 + 轻量内容
使用频次 中低频（有困惑才用） 高频（每晚都可能用）
与书的关系 书是解决方案 书是陪伴者之一

关键洞察：情绪入口不是替代问题入口，而是前置激活层。它让书房从一个"需要时想起的工具"变成"每天都想打开的树洞"。情绪入口带来高频打开，问题入口带来深度留存，两者形成互补。

02

用户画像：600+ 付费会员是谁

第三季会员 + 俱乐部会员，合计约 600 人，第三季人数还在持续增加。这个群体的共同特征是：已经付费、已经信任、已经有一定认知基础。不需要再教育，需要被深度服务。

👤 三类典型用户场景

场景 A：「终于娃睡了，但我睡不着」型

  • 晚上 9-11 点最活跃
  • 不是来"学习"的，是来"喘口气"的
  • 需要的是被理解，不是被指教
  • 对"另一个妈妈也这样"的共情内容最敏感

场景 B：「孩子又磨蹭了，我快炸了」型

  • 情绪爆发后的 30 分钟内
  • 有明确触发事件，但还没冷静下来分析
  • 需要情绪出口 + 事后复盘工具
  • 对"我信息"、"积极倾听"等工具卡片接受度高

场景 C：「今天过得还行，但总觉得少了点什么」型

  • 情绪平稳但略空虚
  • 有自我成长需求，但动力不足
  • 睡前复盘功能的最大受益者
  • 对"小确幸"、"今日觉察"类内容感兴趣
📊 用户行为预判
行为 预估占比 产品启示
每晚打开一次 ~40% 睡前场景是主战场
有情绪时才打开 ~35% 情绪识别速度决定留存
偶尔打开看看 ~20% 推送是拉回的关键
很少打开 ~5% 需要更深层的激活策略
03

情绪入口：不是选标签，是被看见

CC 做的那个心情选择界面，如果能让妈妈感觉到"我现在这个状态被理解了"，那就成功了一半。关键不是给多少选项，而是给的选项能否唤起"被看见"的感觉。

❌ 反模式：固定情绪标签

「焦虑」「生气」「疲惫」「开心」—— 这些词太抽象了。一个妈妈说"我焦虑"，背后可能是孩子磨蹭、老人干预、老公不理解的混合体。标签把丰富的感受压扁了，用户反而觉得"你不懂我"。

✅ 正模式：描述性语句 + 场景感

用一句话描述一个具体场景 + 身体感受，让用户觉得"这说的就是我"。

示例情绪语句：

"孩子又磨蹭了，我快炸了" "终于娃睡了，但我睡不着" "老人又在旁边指指点点" "吼完孩子，现在很后悔" "今天一天像打仗，但不知道忙了什么" "孩子不理我，心里空落落的" "想做个温柔妈妈，但控制不住" "老公又当甩手掌柜了" "觉得自己不是个好妈妈" "今天其实挺顺的，但高兴不起来"

设计原则：每个描述性语句背后，都对应一个或一组"内容推荐策略"。不是随机展示，而是精准匹配。比如"孩子又磨蹭了，我快炸了"背后匹配的是「我信息」工具卡片 + 积极倾听音频片段 + 一个"磨蹭"相关的书中案例。

🎨 入口交互设计

第一层：快速选择（降低门槛）

用户进入"心情"页面，首先看到 6-8 个描述性卡片，横滑或瀑布流展示。每张卡片是一个完整的场景句，带一个与之匹配的小插画（简约线条风）。点击即表示"我现在就是这个状态"。

第二层：自由倾诉（加深连接）

如果快速选择里没有匹配的，用户可以直接输入。输入框的 placeholder 是引导性的：

"用一句话说说此刻发生了什么..."
示例："刚刚陪孩子写作业，说了五遍还是不动，我声音越来越大..."

第三层：情绪强度（辅助判断）

选择完场景后，出现一个极简的情绪温度计：🔥 1-5 级。这个数据用于后续推荐内容的"轻重"——情绪强度高时先给共情，情绪强度低时可以轻推工具。

04

AI 对话流：共情 → 提问 → 轻推 → 邀请

选完心情后，不是直接推书。AI 先聊两句，让用户感觉到"被接住"。这个顺序是产品体验的灵魂。

1

共情 · 被看见

AI 的第一句话必须是共情，不是建议。目标是让用户觉得"你懂我"。不是"我理解你的感受"这种抽象表达，而是具体化她描述的场景。

2

提问 · 引导觉察

共情之后不急着给答案，而是提一个让她稍微停下来想一想的问题。问题要轻，不能让她觉得"你在教育我"。用"我好奇的是..."而不是"你为什么..."

3

轻推 · 一个角度

如果她继续回应，AI 轻轻推一个书中的角度——不是大段解释，而是一个"原来还可以这样想"的瞬间。附带页码引用，引导她翻开书。

4

邀请 · 下一步

最后是一个开放的邀请：继续聊、去听听音频、读读这一章、或者做个小练习。不强迫，给她选择权。

💬 对话示例：「孩子又磨蹭了，我快炸了」

我
孩子又磨蹭了，我快炸了 😤
AI
说了好几遍还是不动，声音越来越大，但孩子像没听见一样……这种无力感，真的让人很挫败。
AI
我好奇的是，在那个瞬间，你最希望发生什么？是孩子立刻拿起笔，还是……别的什么？
我
我就是想让他自觉一点，别什么都让我催
AI
"自觉"——这个词背后，是不是有一点"我希望他能懂我的辛苦"？

《在远远的背后带领》里有一个视角：当我们越界替孩子承担责任（比如不断催促），孩子反而更难发展出自我管理的能力。界限不是不管，而是把"谁的事"理清楚。

📖 第 3 章「中国式界限」P.32 有一个很接近的案例，讲的就是"催作业"这个场景。如果你现在想看，我可以把那一页的金句读给你听。
AI
🎧 或者，如果你现在不想看文字，这里有 3 分钟的一个音频，是另一个妈妈分享她怎么从"催命鬼"变成"守望者"的。要听听吗？
⚠️ 关键红线
  • 不给建议：不说"你应该..."、"你可以试试..."
  • 不说教：不引用教育理论的大段阐述
  • 不急于推荐：至少两轮对话后再轻推内容
  • 不否定情绪：不说"其实没那么严重"、"你想多了"
  • 保持好奇：多用"我好奇的是..."、"是什么让你..."
05

推荐体系：不止于书

如果场域定位是"温暖妈妈的帮助"，那内容形态可以更轻、更多样。核心推荐的不是"一本书"，而是"一个被理解的瞬间"或"一个可以尝试的角度"。书只是其中一种载体。

06

睡前复盘：三句话的温柔

「今天过得怎么样」的睡前复盘——妈妈简单输入三句话，AI 帮她提炼。这个功能的核心价值不是"记录"，而是"帮她在一天结束前，把散落的感受整理成一点点对自己的看见"。

📝 三句话模板

每天晚上，AI 用温柔的语气邀请：

"今天快结束了，想陪你一起回顾一下。不用写很多，三句话就好："

  1. 今天发生了什么？（一件事就行）
  2. 那时候，我身体里最强烈的感受是什么？
  3. 如果可以对那时的自己说一句话，我会说什么？

用户输入后，AI 做两件事：

  • 情绪命名：帮用户把模糊的感受命名，比如"听起来，你今天的关键词是'委屈'和'疲惫'"
  • 温柔提炼：把三句话整理成一段给她的"小纸条"，像朋友的留言

💬 复盘示例

我
1. 今天孩子数学考了78分，我没忍住说了他
2. 看到他低头的时候，我心里很难受
3. 你已经很努力了，是妈妈太急了
AI
💌 给今天的你：

今天的关键词是"着急"和"心疼"。

你看到了分数，第一反应是担心他的未来。但你同时也看见了他低头的样子——那个瞬间，你的心比你的嘴更诚实。

你对自己说的那句话，"是妈妈太急了"，其实已经是你能给的最好的道歉。

明天，如果还有机会，试着把这句话说出来给他听。💛

产品细节：复盘内容可以一键保存到"我的心情日记"，也可以分享（匿名）到社群。每周日晚上，AI 帮用户生成一份"本周情绪小报"，看看这一周自己的心情曲线。

07

连续性：心情 → 推荐的时间线

「心情—推荐这条线需要有连续性。」这意味着用户不是每次进来都是一张白纸，而是有一个属于她的情绪档案。AI 记得她上周为什么难过，也知道她最近在读哪一章。

📈 个人情绪时间线

每个用户有一个私密的"情绪日历"：

  • 每日记录：每次选择/输入的心情状态 + 时间戳
  • 情绪标签云：过去 30 天最常出现的 5 个情绪场景
  • 触发事件追踪：哪些事件反复出现（如"作业"、"老人"、"手机"）
  • 内容消费记录：她看过哪些推荐、听到哪段音频、有没有打开书
🔄 连续性推荐示例

场景：用户上周选择了「孩子磨蹭，我快炸了」，AI 推荐了「我信息」工具卡片。

本周同一场景再次出现：

AI
又是"磨蹭"这个老朋友了。上周我们聊过「我信息」，你还记得那个公式吗——"当你…，我感到…，因为…"

今天想试试不一样的角度吗？或者，如果你已经试过了，告诉我效果如何？

连续性的价值：用户感觉到"这个产品在和我一起成长"，而不是每次都从零开始。

🔔 智能触达：在她需要时出现

基于情绪时间线的数据，可以设计轻度推送：

  • 周末预判：如果用户连续两周在周日晚出现焦虑情绪，周日下午主动推送一段轻音频
  • 深夜守护：晚上 10 点后打开应用，默认进入"睡前模式"（更柔和的色调、更轻的内容）
  • 重复触发预警：同一个场景连续出现 3 次以上，AI 主动建议"要不要聊聊这个模式？"
  • 正向 reinforcement：当用户连续记录 7 天复盘，解锁一句犟爸亲笔签名寄语
08

内容生产与冷启动

600+ 付费会员的内容需求，靠人工写不完，靠纯 AI 生成又没温度。需要一个人机协作的内容生产流水线。

✍️ 内容生产策略
内容类型 生产方式 更新频率
一句话金句 从书中提取 + AI 改写 一次性准备 200 条
书中案例片段 从现有阅读指南提取 已有内容，直接复用
音频陪伴 用 Gemini TTS 生成，人工审校 每周 2-3 条
工具卡片 基于现有工具库生成 已有内容，直接复用
今日小练习 AI 基于场景生成，人工审核 每周 7 条（每日不同）
妈妈来信 征集真实用户投稿 每月 2-3 篇
🚀 冷启动内容库

MVP 阶段需要的最小内容集：

  • 10 个情绪场景 × 3 种内容形态 = 30 条基础推荐内容
  • 5 个核心工具卡片（我信息、积极倾听、第三法、行为窗口、换挡）
  • 10 条 3-5 分钟音频（覆盖最高频的 10 个情绪场景）
  • 7 个今日小练习（一周七天不重样）
  • 50 条一句话金句（兜底内容，情绪强度高时先给）

这个量级可以在 2 周内完成，足够支撑 600 人的冷启动。

09

技术路径与 MVP 优先级

基于书房现有的 Next.js + Prisma + SQLite 架构，情绪入口的落地路径如下：

🏗️ 数据库扩展
// 新增 MoodEntry 表
model MoodEntry {
  id        String   @id @default(uuid())
  userId    String
  moodText  String   // 用户选择的描述语句或输入内容
  moodTag   String   // 归类标签（如"磨蹭","老人","自责"）
  intensity Int      // 1-5 情绪强度
  createdAt DateTime @default(now())
}

// 新增 DailyReflection 表  
model DailyReflection {
  id        String   @id @default(uuid())
  userId    String
  answer1   String   // 发生了什么
  answer2   String   // 身体感受
  answer3   String   // 对自己说
  aiSummary String   // AI 提炼的小纸条
  createdAt DateTime @default(now())
}

// 新增 ContentRecommendation 表（内容库）
model ContentRecommendation {
  id          String @id @default(uuid())
  moodTags    String // JSON: ["磨蹭","作业"]
  contentType String // QUOTE / AUDIO / TOOL / EXERCISE / CASE
  title       String
  body        String // 内容正文或音频 URL
  bookRef     String? // 关联书籍页码
  order       Int    // 排序权重
}

页面 路径 功能

🎯 MVP 分阶段
1

Phase 1：心情入口 + 基础推荐（2 周）

10 个情绪场景 × 3 种内容形态。用户选择心情后，直接展示匹配的内容卡片，暂不涉及 AI 对话。验证"描述性语句"是否比"标签"更有共鸣。

2

Phase 2：AI 共情对话（2 周）

接入现有 AI 教练能力，在推荐内容前增加 2-3 轮对话。验证"被接住"的感觉是否能提升内容打开率。

3

Phase 3：睡前复盘 + 情绪时间线（2 周）

三句话复盘功能上线，同步构建个人情绪档案。验证用户是否愿意持续记录，以及连续性推荐的效果。

4

Phase 4：社群化内容 + 智能触达（持续）

妈妈来信征集、每周情绪小报、智能推送。验证社群内容是否能提升打开频次。
